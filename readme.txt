Примерная нагрузка:
5М * 0.2 = 1М юзеров с подпиской
1М / 12 ~ 84000 юзеров в месяц
84000 / 30 ~ 3000 емейлов в день

Можно разослать с одной машины в 10 потоков примерно за 5 часов. Естественно, можно количеество потоков увеличить.
Для одной машины все упирается скорее в производительность postfix/sendmail.

Если мы хотим использовать несколько машин, то нужно модифицировать скрипт reminderQueueManager.
Каждый инстанс должен брать из очереди записи по остатку от деления на количество инстансов. (id % instancesTotal = instanceN)

Crontab:
0 0 * * * php fillReminderQueue.php 1 >> /local/logs/fillReminderQueue.log 2>&1
0 0 * * * php fillReminderQueue.php 3 >> /local/logs/fillReminderQueue.log 2>&1
0 * * * * php reminderQueueManager.php >> /local/logs/reminderQueueManager.log 2>&1

---
Логика работы:
1) скрипт fillReminderQueue.php наполняет очередь отправки, интервал в днях передается первым параметром. Запускается раз в сутки.
2) скрипт reminderQueueManager.php это многопоточный скрипт, менеджер очереди. Работает до тех пор, пока есть что рассылать.
   Запускается каждый час, чтобы обрабатывать записи с ошибками. Можно запускать чаще.
3) скрипт reminderQueueProcessor.php собственно и занимается рассылкой. Вызывается из менеджера.
